# Повторный экзамен #2: Сравнительный обзор 3 сущностей (Tavily)

## Requirements
- [high] Определение состояния: Создать TypedDict CompareState с полями entities, criteria, findings, final_table, verdict.
- [high] Нод plan_criteria: LLM генерирует 3–5 критериев сравнения на основе заданных сущностей.
- [high] Нод research_entity: Пошагово обрабатывает каждую пару entity×criterion, выполняет Tavily‑поиск и сохраняет краткую заметку в findings.
- [high] Нод build_table: Собирает Markdown‑таблицу из findings, где строки – критерии, колонки – сущности.
- [high] Нод verdict: LLM формирует 2–4 предложения с рекомендацией по выбору сущности для конкретного кейса.
- [high] Циклическая логика графа: Пока есть необработанные пары entity×criterion – повторять research_entity, иначе переходить к build_table.
- [high] Интеграция Tavily: Использовать langchain‑tavily или Tavily SDK для реального веб‑поиска, не генерируя фиктивные ссылки.
- [high] CLI демо: Разработать командную строку, позволяющую вводить собственные три сущности или использовать тему по умолчанию, выводить план критериев, найденные пары и итоговую таблицу с вердиктом.
- [normal] README и запуск: Подготовить README с инструкциями по установке зависимостей, настройке .env и запуску скрипта, убедиться, что проект запускается без ошибок.
